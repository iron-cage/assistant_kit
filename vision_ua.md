# assistant — Візія

Claude Code — найпотужніший AI-асистент для розробки. При цьому в нього немає публічного
API, немає програмного інтерфейсу, і немає жодного способу інтегрувати його в автоматизовані
процеси без написання парсерів файлів і shell-викликів з нуля. Кожна команда, яка серйозно
ним користується, рано чи пізно пише один і той самий клей-код.

`assistant` це змінює.

## Простір проблем

Три точки тертя виникають при серйозній роботі з Claude Code:

**Ротація облікових записів — ручна і ненадійна.** Кожна підписка має п'ятигодинне вікно
активного використання. Переключення між підписками означає ручне редагування
`~/.claude/.credentials.json` — один збій запису, і сесія зникає.

**Життєвий цикл сесій — непрозорий.** Історія розмов зберігається у `~/.claude/projects/`
з неочевидним екранованим ключем. Відповідь на питання «продовжити чи почати заново?»
вимагає ручного перегляду недокументованих структур директорій.

**Автоматизація потребує саморобного клею.** Будь-який інструмент, що викликає Claude Code
— CI-хук, бот для ревʼю, завдання за розкладом — має знати, які прапорці передавати, як
обробляти інтерактивний і неінтерактивний режими, як підключити stderr і stdout.
Задокументованої угоди про виклики немає.

## Що ми побудували

Шаруватий Rust-workspace із 13 крейтів:

```
Шар 3: assistant              Супер-застосунок, що агрегує всі CLI-інструменти (clt)
             ↓
Шар 2: claude_profile  (clp)  Управління обліковими записами, статус токенів, шляхи ~/.claude/
        claude_storage  (clg)  CLI для дослідження сховища сесій Claude Code
        claude_runner   (clr)  Виконання Claude Code із підтримкою продовження сесій
        claude_version  (clv)  Встановлення, версії, сесії та управління налаштуваннями
        claude_assets   (cla)  Встановлення артефактів (правила, навички, команди) через симлінки
        dream                  Бібліотечний фасад, що ре-експортує всі core-крейти
             ↓
Шар 1: claude_profile_core    Статус токенів + доменна логіка облікових записів
        claude_version_core    Виявлення версій, допоміжники налаштувань
        claude_runner_core     Builder для ClaudeCommand + єдина точка виконання процесів
        claude_assets_core     Доменна логіка інсталятора артефактів через симлінки
             ↓
Шар 0: claude_common          Спільні примітиви: ClaudePaths, утиліти процесів
*       claude_storage_core    JSONL-парсер без залежностей для ~/.claude/
```

## Принципи проєктування

**Атомарні операції.** Переключення облікових записів використовує write-then-rename, а не
перезапис. Збій у будь-який момент залишить або старі, або нові облікові дані — ніколи не
напівзаписаний файл.

**Одне місце для кожного рішення.** `ClaudePaths` — єдине авторитетне джерело всіх шляхів
`~/.claude/`. `ClaudeCommand::execute()` — єдина точка запуску процесів. Обидва підкріплені
тестами, що точно вкажуть, який файл порушив правило.

**Документація — джерело істини.** Кожен крейт постачається з директорією `docs/` з
екземплярами feature, invariant і pattern doc. Кожен пункт контрольного списку має
іменований тест, який повинен проходити, щоб галочка залишалася.

## Поточний стан

13 крейтів. Усі проходять L3 (nextest + doc tests + clippy, нуль попереджень).

Перевірено в продуктиві:
- `claude_storage` парсить ~1 900 проєктів і 2 400 сесій (~7 ГБ JSONL) менш ніж за
  30 секунд із теплим кешем
- Виявлення продовження сесій коректно працює в живих інструментах
- `claude_runner` автоматизує багатокрокові ревʼю

## Відкриті завдання

- **Автоматизація ротації:** Демон, що стежить за `TokenStatus` і ротує облікові записи
  при `ExpiringSoon` — деталь, що робить роботу з кількома підписками справді безперервною.
- **Аналітика використання:** Витрати токенів за проєктом, днем і типом розмови на основі
  наявної статистики `claude_storage_core`.
- **Відтворення розмов:** JSONL-формат повністю розпарсений — вилучення, підсумовування
  і порівняння між сесіями можна легко побудувати.
- **Інтеграція з CI:** `claude_runner` на pull request'ах як структурований крок ревʼю
  з виводом, придатним для парсингу і публікації як коментар.
- **Публікація на crates.io:** Інваріант приватності вже забезпечено. Публікація крейтів
  Шарів 0 і 1 — логічний наступний крок після стабілізації інтерфейсів.

## Стек

Rust 2021, педантичний clippy із `missing_inline_in_public_items` і `std_instead_of_core`
як жорсткими помилками. Кожен публічний елемент задокументований. Кожна функціональна
вимога прив'язана до іменованого тесту.
