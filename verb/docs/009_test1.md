# Verb: `test1`

- **Kind:** canonical
- **Availability:** universal
- **`--dry-run`:** `./runbox/runbox .test1 '<filter>'`

### Command

```bash
./verb/test1 <nextest-filter>
```

Delegates to `runbox/runbox`, which builds the Docker image if needed and runs only the matching nextest filter inside the container. The container executes `verb/test.d/l1_filter`, which runs `RUSTFLAGS="-D warnings" cargo nextest run --all-features -E "<filter>"`. No doc tests or clippy ā€” use `./verb/test` for the full suite.

### Layers

| Layer | Context | Docker | `CARGO_NET_OFFLINE` | Default |
|-------|---------|--------|---------------------|---------|
| runbox | host ā†’ Docker via `runbox/runbox .test1` | yes | yes (inside container) | yes ā€” only path |

### Notes

`verb/test1 <filter>` is the **mandatory** choice for targeted verification. Running `./verb/test` (full suite) to check a single test is a performance violation per `rulebook.md Ā§ Test Execution : Container-Only Testing` ā€” `verb/test1` avoids nextest full-suite + clippy + doc-test overhead.

The filter is a nextest filter expression passed as `NEXTEST_FILTER` env var to the container. `verb/test.d/l1_filter` reads it and runs `cargo nextest run --all-features -E "$NEXTEST_FILTER"`.

`--dry-run` (first argument) prints `./runbox/runbox .test1 '<filter>'` and exits 0 ā€” no tests run.

### Example

```bash
# Run a single test by exact name:
./verb/test1 test_config_get_existing_key

# Run all tests whose name contains a prefix:
./verb/test1 'test_config_'

# Dry run (prints command, no execution):
./verb/test1 --dry-run
```

`verb/test1` script:
```bash
#!/usr/bin/env bash
# test1 ā€” run a single nextest filter inside the container.
set -euo pipefail
FILTER="${1:-}"
if [[ -z "$FILTER" ]]; then
  echo "error: usage: ./verb/test1 <nextest-filter>" >&2
  echo "  example: ./verb/test1 test_config_get" >&2
  exit 1
fi
if [[ "$FILTER" == "--dry-run" ]]; then echo "./runbox/runbox .test1 '<filter>'"; exit 0; fi
exec "$(dirname "${BASH_SOURCE[0]}")/../runbox/runbox" .test1 "$FILTER"
```

`verb/test.d/l1_filter` (container-internal; entered via `NEXTEST_FILTER` env var set by `runbox-run`):
```bash
#!/usr/bin/env bash
# l1_filter ā€” targeted single-filter test execution inside the container.
set -euo pipefail
SCRIPT_DIR="$( cd "$( dirname "${BASH_SOURCE[0]}" )" && pwd )"
cd "$SCRIPT_DIR/../.."
FILTER="${NEXTEST_FILTER:-}"
if [[ -z "$FILTER" ]]; then
  echo "error: NEXTEST_FILTER env var not set" >&2; exit 1
fi
export CARGO_NET_OFFLINE=true
export CARGO_TARGET_DIR=/workspace/target
export NO_COLOR=1
exec RUSTFLAGS="-D warnings" cargo nextest run --all-features -E "$FILTER"
```
