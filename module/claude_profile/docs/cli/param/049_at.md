# Parameter :: 49. `at::`

Sets the billing renewal override timestamp to an absolute ISO-8601 UTC datetime. The value is stored as `_renewal_at` in `{name}.json`. Past values are accepted and auto-advanced monthly by `.usage` at render time.

- **Type:** `string` — ISO-8601 UTC datetime
- **Default:** *(omit; required when neither `from_now::` nor `clear::1` is provided)*
- **Constraints:** Must be a parseable ISO-8601 datetime with UTC offset (`Z` or `+00:00`); date-only form (`YYYY-MM-DD`) accepted as `T00:00:00Z`
- **Mutually exclusive with:** `from_now::`, `clear::`
- **Commands:** [`.account.renewal`](../command/001_account.md#command--14-accountrenewal)
- **Purpose:** Pin an exact billing renewal date/time for an account, correcting the `org_created_at`-based estimate shown in `.usage`.

**Valid formats:**
- `2026-06-29T21:00:00Z`
- `2026-06-29T21:00:00+00:00`
- `2026-06-29` (midnight UTC)

**Usage:**

```bash
clp .account.renewal name::alice@acme.com at::2026-06-29T21:00:00Z
clp .account.renewal name::all at::2026-05-29T21:00:00Z
clp .account.renewal name::alice@acme.com at::2026-06-29T21:00:00Z dry::1
```
