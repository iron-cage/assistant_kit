# Parameter :: 53. `for::`

Specifies the target host+user identity for the `.account.assign` command as a single `USER@MACHINE` composite value. The command writes the active-account marker file for that identity instead of the current machine.

- **Default:** current machine's `$USER@resolve_hostname()` (same sources as `active_marker_filename()` ‚Äî see [feature/025](../../feature/025_per_machine_active_marker.md))
- **Constraints:** When provided, must contain at least one `@`; both the user part (left of first `@`) and the machine part (right of first `@`) must be non-empty after splitting. Each component is sanitized: alphanumeric, `-`, `.` kept; all other characters become `_`.
- **Purpose:** Write the per-machine active-account marker for a target machine without being logged in there ‚Äî enables pre-seeding account preferences in a shared or synced credential store.

**Value format:** `USER@MACHINE`

Split on the **first** `@`:
- Left of `@` ‚Üí user component ‚Üí sanitized ‚Üí second segment of `_active_{machine}_{user}`
- Right of `@` ‚Üí machine component ‚Üí sanitized ‚Üí first segment of `_active_{machine}_{user}`

The `@` separator mirrors SSH/remote convention (`user@host`) ‚Äî unambiguous and familiar.

**Sanitization:** same char-filter as `active_marker_filename()`: alphanumeric, `-`, and `.` kept; all other characters become `_`.

**Examples:**

| `for::` value | Written filename |
|---------------|-----------------|
| `alice@laptop` | `_active_laptop_alice` |
| `user1@w003.local` | `_active_w003.local_user1` |
| `alice@my laptop` | `_active_my_laptop_alice` (space ‚Üí `_`) |
| (omitted) | `_active_{hostname}_{user}` ‚Äî current machine default |

**Usage:**

```bash
clp .account.assign name::work@corp.com for::alice@laptop
clp .account.assign name::work@corp.com for::user1@w003
clp .account.assign name::work@corp.com for::alice@laptop dry::1
```

**Error cases:**
- `for::badvalue` (no `@`) ‚Üí exit 1 with message explaining `USER@MACHINE` format
- `for::@laptop` or `for::alice@` (empty component) ‚Üí exit 1

**See Also:** [feature/032_account_assign.md](../../feature/032_account_assign.md) ‚Äî full command semantics and execution steps.
