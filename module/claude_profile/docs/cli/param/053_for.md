# Parameter: 53. `for::`

> **REMOVED (Feature 064):** The `for::` parameter has been removed from the unified parameter set on `.accounts` and `.usage`. Its functionality was absorbed into the `active::` parameter (Feature 064), which was then renamed to `assignee::` (Feature 065). Current CLI surface uses `assignee::`.
>
> **Migration:**
> - `assign::1 name::X for::USER@MACHINE` â†’ `assignee::USER@MACHINE name::X`
> - `assign::1 name::X` (current machine) â†’ `assignee::0 name::X`
>
> Using `for::` now exits 1 with a migration message.
>
> See [feature/065_assignee_param_redesign.md](../../feature/065_assignee_param_redesign.md) for full context.

[Historical specification retained below for reference.]

---

Specifies the target host+user identity for the `.account.assign` command as a single `USER@MACHINE` composite value. The command writes the active-account marker file for that identity instead of the current machine.

- **Default:** current machine's `$USER@resolve_hostname()` (same sources as `active_marker_filename()` â€” see [feature/025](../../feature/025_per_machine_active_marker.md))
- **Constraints:** When provided, must contain at least one `@`; both the user part (left of first `@`) and the machine part (right of first `@`) must be non-empty after splitting. Each component is sanitized: alphanumeric, `-`, `.` kept; all other characters become `_`.
- **Purpose:** Write the per-machine active-account marker for a target machine without being logged in there â€” enables pre-seeding account preferences in a shared or synced credential store.

**Value format:** `USER@MACHINE`

Split on the **first** `@`:
- Left of `@` â†’ user component â†’ sanitized â†’ second segment of `_active_{machine}_{user}`
- Right of `@` â†’ machine component â†’ sanitized â†’ first segment of `_active_{machine}_{user}`

The `@` separator mirrors SSH/remote convention (`user@host`) â€” unambiguous and familiar.

**Sanitization:** same char-filter as `active_marker_filename()`: alphanumeric, `-`, and `.` kept; all other characters become `_`.

**Examples:**

| `for::` value | Written filename |
|---------------|-----------------|
| `alice@laptop` | `_active_laptop_alice` |
| `user1@w003.local` | `_active_w003.local_user1` |
| `alice@my laptop` | `_active_my_laptop_alice` (space â†’ `_`) |
| (omitted) | `_active_{hostname}_{user}` â€” current machine default |

**Usage:**

```bash
clp .account.assign name::work@corp.com for::alice@laptop
clp .account.assign name::work@corp.com for::user1@w003
clp .account.assign name::work@corp.com for::alice@laptop dry::1
```

**Error cases:**
- `for::badvalue` (no `@`) â†’ exit 1 with message explaining `USER@MACHINE` format
- `for::@laptop` or `for::alice@` (empty component) â†’ exit 1
