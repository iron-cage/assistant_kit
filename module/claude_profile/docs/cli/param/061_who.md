# Parameter: `who::`

### Scope

- **Purpose**: Control visibility of the sessions table in `.usage` output — a `data_fmt` table showing all `_active_*` markers in the credential store as `{user}@{host} → account`.
- **Responsibility**: Documents the `who::` parameter accepted by `.usage`.

### Design

The sessions table surfaces which host/user sessions are active across the credential store. Each `_active_{host}_{user}` marker file is read; the filename is parsed into a `{user}@{host}` identity and the file content provides the account name. The current machine's own session receives `✓` in the Account column.

**Default behavior (auto):** The sessions table is shown when >1 `_active_*` marker exists in the credential store, hidden when ≤1 (single-session case adds no information). The `who::` parameter overrides this default:

| Value | Behavior |
|-------|----------|
| *(omit)* | Auto: shown when >1 marker, hidden when ≤1 |
| `1` | Force sessions table on (even when ≤1 marker) |
| `0` | Force sessions table off (even when >1 marker) |

### Specification

| Attribute | Value |
|-----------|-------|
| Parameter | `who::` |
| Type | `bool (0 or 1)` |
| Default | auto (>1 marker → shown) |
| Valid Values | `0`, `1` |
| Commands | `.usage` |
| Pipeline Stage | Display |
| Group | [Display Control](../param_group/005_display_control.md) |

### Acceptance Criteria

- **AC-01**: `who::0` suppresses the sessions table regardless of marker count.
- **AC-02**: `who::1` forces the sessions table even when ≤1 marker exists.
- **AC-03**: When `who::` is omitted, the sessions table is shown when >1 `_active_*` marker exists and hidden when ≤1.
