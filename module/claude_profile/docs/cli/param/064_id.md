# Parameter: 64. `id::`

Pins the subprocess model for clr task-execution commands by setting `subprocess_model` in `~/.clr/prefs.json`. Present on `.model.select` only. When provided, activates set mode; when absent, `.model.select` operates in read (get) mode unless `reset::1` is set.

- **Default:** *(omit)* — get mode; reads current preference from `~/.clr/prefs.json`
- **Constraints:** Any non-empty string; expected to be a full model ID (e.g., `claude-opus-4-8`)
- **Purpose:** Write a full model ID to `~/.clr/prefs.json` for use by `clr run`, `clr ask`, `clr isolated`, and `clr refresh`.

**Values:**

Any full model ID returned by `clp .models` is accepted. Common values:

| Value | Effect |
|-------|--------|
| `claude-opus-4-8` | Pin to Opus 4.8 for all clr task subprocesses |
| `claude-sonnet-5` | Pin to Sonnet 5 for all clr task subprocesses |
| `claude-haiku-4-5-20251001` | Pin to Haiku 4.5 for all clr task subprocesses |

**Validation:** Non-empty check at write time only. Model validity is not checked against the live API at pin time — the Claude API will reject an invalid ID when clr invokes the subprocess.

**Error cases:**
- `id::` (empty value) → exit 1; stderr: `id:: must be a non-empty model ID`
- `id::VALUE reset::1` together → exit 1; stderr: `id:: and reset::1 are mutually exclusive`

**Examples:**

```bash
clp .model.select id::claude-opus-4-8   # pin Opus 4.8
clp .model.select id::claude-sonnet-5   # pin Sonnet 5
```

**Notes:**
- `id::` governs the subprocess model for `clr run/ask/isolated/refresh`. For the interactive Claude Code session model, use `clp .model set::opus` instead.
- Run `clp .models` first to discover available full model IDs.
- The pinned preference persists until explicitly reset with `clp .model.select reset::1`.
