# JSONL: Usage Object

### Scope

- **Purpose**: Specify the token usage tracking object within assistant message entries.
- **Responsibility**: Authoritative instance for `message.usage` — all token count fields, cache tracking, service tier.
- **In Scope**: `input_tokens`, `cache_creation_input_tokens`, `cache_read_input_tokens`, `cache_creation` sub-object, `output_tokens`, `service_tier`.
- **Out of Scope**: Usage in endpoint API responses (→ `../endpoint/003_v1_messages.md`); assistant entry envelope (→ [003_assistant_entry.md](003_assistant_entry.md)).

### Schema

```json
{
  "input_tokens": 9,
  "cache_creation_input_tokens": 10156,
  "cache_read_input_tokens": 12112,
  "cache_creation": {
    "ephemeral_5m_input_tokens": 10156,
    "ephemeral_1h_input_tokens": 0
  },
  "output_tokens": 6,
  "service_tier": "standard"
}
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `input_tokens` | number | Regular input tokens (uncached) |
| `cache_creation_input_tokens` | number | Tokens used to create cache entries |
| `cache_read_input_tokens` | number | Tokens read from existing cache (billed at lower rate) |
| `cache_creation.ephemeral_5m_input_tokens` | number | Tokens in 5-minute ephemeral cache tier |
| `cache_creation.ephemeral_1h_input_tokens` | number | Tokens in 1-hour ephemeral cache tier |
| `output_tokens` | number | Tokens generated by the assistant |
| `service_tier` | string | API service tier: `"standard"` |

### Notes

**Cache tracking**: Added in Claude Code v2.0.25+. `cache_creation` sub-object is optional — absent in entries from older versions.

**Total cost calculation**: `input_tokens + cache_creation_input_tokens` = total tokens billed at input rate; `cache_read_input_tokens` billed at cache-read rate (significantly lower).

**`cache_creation` vs `cache_creation_input_tokens`**: Both fields track the same tokens. `cache_creation_input_tokens` is the top-level summary; `cache_creation` breaks it down by cache tier duration.
