# Subcommand: mcp

Configure and manage MCP servers.

### Usage

```
claude mcp [command]
```

### Sub-subcommands

| Command | Description |
|---------|-------------|
| `add [options] <name> <commandOrUrl> [args...]` | Add an MCP server (stdio or HTTP) |
| `add-from-claude-desktop [options]` | Import MCP servers from Claude Desktop (Mac/WSL only) |
| `add-json [options] <name> <json>` | Add an MCP server with a JSON config string |
| `get <name>` | Get details about a configured MCP server |
| `list` | List all configured MCP servers |
| `remove [options] <name>` | Remove an MCP server |
| `reset-project-choices` | Reset approved/rejected project-scoped (.mcp.json) servers |
| `serve [options]` | Start the Claude Code MCP server (expose Claude Code as MCP) |

### Description

Full MCP (Model Context Protocol) server lifecycle management. Supports adding
servers via command-line arguments, JSON configuration, or import from Claude
Desktop. Servers can use stdio or HTTP transport.

Examples:
```
# Add HTTP server:
claude mcp add --transport http sentry https://mcp.sentry.dev/mcp

# Add stdio server with env vars:
claude mcp add -e API_KEY=xxx my-server -- npx my-mcp-server

# Import from Claude Desktop:
claude mcp add-from-claude-desktop
```

The `serve` sub-subcommand starts Claude Code itself as an MCP server,
allowing other MCP clients to connect to it.
