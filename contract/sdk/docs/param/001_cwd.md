# cwd

Working directory the spawned `claude` subprocess operates in.

### Forms

| | Value |
|-|-------|
| TS Field | `cwd?: string` |
| Python Field | `cwd` (inferred snake_case-identical; not independently confirmed) |
| CLI Equivalent | — (CLI inherits the shell's cwd; no dedicated flag) |

### Type

string (filesystem path)

### Default

`process.cwd()` (TypeScript) — the SDK caller's own current working directory at call time.

### Since

SDK GA

### Description

Sets the working directory the spawned `claude` subprocess is launched in — directly analogous to a `Command::current_dir()` call a Rust bridge would make when spawning `claude` itself (see [`../pattern/002_rust_bridge_strategies.md`](../pattern/002_rust_bridge_strategies.md)). Because tool calls like `Read`/`Write`/`Bash` resolve relative paths against the subprocess's own cwd, not the SDK caller's, this field is effectively how a multi-project host application scopes an agent to one project's directory — the CLI has no equivalent flag because an interactively- or `clr`-launched `claude` process simply inherits its parent shell's cwd already.
